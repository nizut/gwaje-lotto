# 컨테이너 구조 초안

## 1. 목적
로또 사이트를 2개 이상의 Docker 컨테이너로 나누어 실행하기 위한 구조를 정리한다.

## 2. 기본 컨테이너 구성
### 1) web 컨테이너
- Django 애플리케이션 실행
- 사용자 요청 처리
- 관리자 페이지 처리
- 정적 파일 제공 준비

### 2) db 컨테이너
- 데이터 저장
- 구매 정보, 회차 정보, 당첨 결과 저장
- 웹 컨테이너와 네트워크로 연결

## 3. 연결 관계
- 사용자는 web 컨테이너로 접속한다.
- web 컨테이너는 db 컨테이너에 연결해 데이터를 읽고 쓴다.
- 환경변수로 DB 주소, 계정 정보, SECRET_KEY 등을 전달한다.

## 4. docker-compose 역할
- 컨테이너를 한 번에 실행
- 컨테이너 간 네트워크 연결 관리
- 환경변수 주입
- 실행 순서와 의존성 관리

## 5. 배포 시 확인할 항목
- 웹 컨테이너가 정상적으로 시작되는지
- DB 컨테이너가 정상적으로 연결되는지
- 마이그레이션이 실행되는지
- 정적 파일이 정상적으로 처리되는지
- 관리자 페이지와 사용자 페이지가 분리되는지
