# 3일 시간표

## 1일차: 요구사항 정리와 개발 준비
- 로또 사이트 요구사항 정리
- 화면 흐름 정리: 일반 사용자 구매, 당첨 확인, 관리자 판매 내역, 추첨, 당첨 내역
- 데이터 구조와 컨테이너 구조 초안 설계
- Django 프로젝트와 앱 구성 준비
- Dockerfile과 docker-compose.yml 초안 작성
- DB, 템플릿, 정적파일, 환경변수 기본 흐름 정리
- 보고서 목차 초안 작성

## 2일차: 사이트 구현과 컨테이너화
- 수동 번호 구매 기능 구현
- 자동 번호 구매 기능 구현
- 당첨 확인 기능 구현
- 관리자용 판매 내역 조회 구현
- 관리자용 추첨 기능 구현
- 관리자용 당첨 내역 조회 구현
- 웹 컨테이너와 DB 컨테이너 분리
- Docker Compose로 실행 구조 연결
- 환경변수와 마이그레이션 설정
- 전체 실행 확인

## 3일차: 테스트와 제출 정리
- 수동 구매 시나리오 테스트
- 자동 구매 시나리오 테스트
- 추첨 및 당첨 확인 테스트
- Docker Compose로 전체 서비스 재실행 테스트
- 오류 수정 및 최종 점검
- 시스템 설계, 구현 과정, 테스트 결과 정리
- AI 도구 사용 내역 정리
- GitHub 링크 포함해 보고서 완성
