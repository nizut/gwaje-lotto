# 데이터 구조 초안

## 1. 목적
로또 사이트에서 구매 정보, 회차 정보, 당첨 번호, 당첨 결과를 저장하기 위한 데이터 구조를 정리한다.

## 2. 주요 데이터
- 사용자 구매 정보
- 로또 번호 정보
- 회차 정보
- 당첨 번호 정보
- 당첨 결과 정보
- 관리자 추첨 기록

## 3. 테이블 초안
### 1) LotteryRound
- 회차 번호
- 추첨일
- 당첨 번호
- 보너스 번호
- 추첨 여부

### 2) Purchase
- 구매 번호
- 구매 시간
- 구매 방식(수동 / 자동 / 반자동)
- 구매 수량
- 연결된 회차

### 3) LottoTicket
- 티켓 번호
- 6개 번호
- 구매와의 연결 정보
- 당첨 여부

### 4) WinningResult
- 결과 번호
- 회차 정보
- 구매 정보
- 당첨 등수
- 당첨 금액

### 5) DrawHistory
- 추첨 기록 번호
- 관리자 정보
- 회차 정보
- 추첨 시간
- 추첨 결과

## 4. 관계 초안
- Purchase 1건에는 여러 개의 LottoTicket이 연결된다.
- LotteryRound 1개에는 1개의 당첨 번호 세트가 연결된다.
- WinningResult는 특정 Purchase와 특정 LotteryRound를 연결한다.
- DrawHistory는 추첨 수행 기록을 남긴다.

## 5. 정리 포인트
- 번호는 1~45 범위의 중복 없는 6개 숫자로 저장한다.
- 수동/자동/반자동 구매 방식을 구분한다.
- 회차별로 구매와 당첨 결과를 추적할 수 있게 설계한다.
